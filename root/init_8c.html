Descripción detallada

Rutinas de inicialización.

En este archivo están las rutinas para inicializar los distintos módulos que componen el sistema. Los mas destacados son los timers, el SPI y PWM. Aquí también se encuentra la inicialización de la variables del sistema (PID, eeprom, I/O,etc).

Inicializa las variables del sistema (pid, pines, configuracion).

La inicialización consiste en asignar condiciones iniciales a la parte integrativa y derivativa del pid, setear los leds al estado correspondiente, encender la fuente y leer de la eeprom las variables de configuración seleccionadas previamente en el menú.

void PWM_init ( int  ciclo_util  ) 

Configuración del PWM.

El PWM se genera con el módulo CCP1. Sólo tres registros se configuran, y retorna generando la señal en el pin P1.1. Posteriormente, el duty cycle se actualiza simplemente cargando el valor correspondiente en CCP1H.

void spi_init ( void   ) 

Configuración del módulo SPI.

La comunicación SPI se configura en modo (0,0), como master, y deshabilitando el Slave Select (), pues en nuestro hardware el pin no está llevado a cero.

De esta forma, obtenemos 13kbit/sec, y una transacción de 3 bytes para muestrear un dato requiere unos 250uSec.

Se habilitan las interrupciones, para que el protocolo sea manejado en el ISR, y el SPI se deja andando.

void timer0_init ( void   ) 

Configuración del timer 0.

El timer 0 es configurado en modo 0, autorecargable, habilitando la interrupción. El trigger es interno y lo deja andando.
